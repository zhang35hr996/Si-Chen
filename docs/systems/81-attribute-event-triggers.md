# Attribute → Event Trigger Table

属性高低(或组合)**触发哪些事件**。这是设计三张表的第二张:
[`21`](21-attribute-catalog.md)=名词(有哪些属性)→ [`80`](80-attribute-action-flow.md)=动词(行动如何改属性)
→ 本表=后果(属性状态触发什么事件)。事件的**优先级/冷却/选取机制**见
[`50-event-trigger-rules.md`](50-event-trigger-rules.md);派系压力详见 [`70`](70-factions-and-pressure.md)。

## 通则

- 阈值带沿用 [`21`](21-attribute-catalog.md):**偏低 ≤39 / 中 40–60 / 偏高 ≥61**;极端用 ≤19 / ≥81。
- **组合触发**(两属性同时满足)是"古代王朝感"的主菜,优先级应高于单属性触发。
- 同名压力源在 [`80`](80-attribute-action-flow.md) 有对应**调节手段**——事件不是死局,是给玩家施压。

## Implementation status

**Design only**。当前事件系统([`50`](50-event-trigger-rules.md))已实现 checkpoint/priority/cooldown,
但本表依赖的属性(皇权安全/暴戾/宗室不满/外戚权势/母家忠心… )尚未进 `Resources`/character state。
落地顺序建议:先加属性字段([`80`](80-attribute-action-flow.md) status),再 author 这些 trigger。

## 一、皇帝 / 皇权

| 触发条件 | 事件 | 主要后果 |
|---|---|---|
| 皇权安全 ≤19 | 逼宫 / 政变 / 废帝 / 王朝覆灭 | **终局**或重大转折 |
| 皇权安全 ≤39 **+** 宗室不满 ≥61 | 宗室拥立旁支 | 皇权安全↓↓、内战风险 |
| 暴戾 ≥61 | 侍君恐惧普涨、暗害/出逃 | 情意↓、后宫背叛风险↑ |
| 暴戾 ≥61 **+** 大臣真实忠心 ≤39 | 朝臣阳奉阴违 | 奏折失真↑、朝政效率暗降 |
| 暴戾 ≥81 | 酷烈之政、民变 | 民心↓↓、宗室不满↑、谣言热度↑ |
| 健康 ≤39 **或** 疲劳 ≥61 | 病倒 | 行动点↓、传位压力事件 |
| 健康 ≤19 | 寿命警告 / 弥留 | 立储/托孤剧情 |
| 威望 ≥61 | 朝野畏服 | 逼宫概率↓、任官/奏折顺利 |
| 勤政 长期 ≤39 | 朝纲松弛 | 朝政滑落、贪腐滋长提示 |

## 设计自检

每个**关键压力源**都应满足"有进有出":本表给"出"(触发事件),
[`80`](80-attribute-action-flow.md) 给"进"(调节手段)。落地前逐条核对:
表里每个 ≥61/≤39 触发,在 80 表都能找到至少一个反向调节动作,否则就是死局。
