## Global Constraints

1. **唯一装配点**：四字段只在 `assembleDialogueRequest` 填充；`compilePromptPayload` 纯透传零改动。
2. **Typed ContextRef**：所有 context 引用使用 `{ kind: "memory" | "event"; id: string }`，不用裸字符串集合。Writeback 按 `kind` 分流：只有 `kind === "memory"` 进 `MemoryMentionRecord`。
3. **Fact key 不含 modality**：`[predicate, subjectId, object ?? null]` — `claimFactKey()` 函数集中定义。Modality 另做强度覆盖（`rumor(0) ≤ suspect(1) ≤ assert(2)`）；`deny` = 相反极性（object 取反），不是弱模态。
4. **Empty allowedClaims = CLOSED**：`policy.allowedClaims` 非 undefined（即使空数组）时，任何 factual `ProposedClaim` 无覆盖则 `claim_not_allowed`。问候/情绪/礼貌话不是 factual claim，不受此规则约束。
5. **Forbidden 语义 = "禁止做出的 claim"**：forbidden 存储实际禁止断言的 claim（object = 禁止的真值），不是"纠正事实"。gate 比较时 fact key 必须与 proposed claim 精确匹配，polarity 一致。
6. **Event provenance 全链路**：`selectReactionEvent` → 返回 `{ event, reactionContext }` → `buildReactionPlan` 保留 `sourceEventId` → `resolveClaimSources` 把 event id 绑定到具体 claim 的 `sourceRefs`。
7. **Retrieval：event 独立 quota**：`buildMemoryContext` 增 `topEvents: number`（默认 3）+ recall query 传 `targetId`/`speakerId`/`audienceCharacterIds`，防 top-5 全被 memory 占满。
8. **Event cooldown**：`state.eventReactionLog: EventReactionRecord[]`；selector 过滤 `MAX_REACTION_COOLDOWN_DAYS`（= 3）内已对同一 audience 反应过的 event；成功 reaction 写 record；save schema v8。
9. **`toPromptEvent(e, db)`**：per-event-type payload whitelist + participant display name 注入；禁止通用 scalar filter。
10. **文件正确性**：claim gate → `src/engine/dialogue/claimGate.ts`；接口定义 → `src/engine/dialogue/types.ts`；不改 `gates.ts`（text gate，另一模块）。
11. **纯函数 / 确定性**：所有新派生函数禁 `Math.random()` / `Date()`；tie-break 稳定排序。
12. **测试**：每个 task 结束须 `npx vitest run` 全绿 + `npx tsc --noEmit` clean。

```ts
// src/engine/dialogue/types.ts（追加）

/** Typed reference to an offered context item (memory or event). */
export interface ContextRef {
  kind: "memory" | "event";
  id: string;
}

/** Stable key for de-duplicating context refs in sets. */
export function contextRefKey(ref: ContextRef): string {
  return `${ref.kind}:${ref.id}`;
}

/**
 * A claim the engine has explicitly authorized (allowed or forbidden).
 * source refs list which memory/event IDs authorize this claim.
 * certaintyCeiling caps how confidently the speaker may assert it.
 */
export interface AuthorizedClaim {
  claim: DialogueClaim;
  sourceRefs: ContextRef[];        // must be non-empty
  certaintyCeiling?: number;       // 0–1; if present, proposed certainty must not exceed
}

/**
 * Fact key for semantic comparison — intentionally excludes modality and claim id.
 * [predicate, subjectId, object ?? null]
 */
export function claimFactKey(
  c: Pick<DialogueClaim, "predicate" | "subjectId" | "object">,
): string {
  return JSON.stringify([c.predicate, c.subjectId, c.object ?? null]);
}

/** Modality strength ordering (deny is opposite polarity, not strength). */
export const MODALITY_STRENGTH: Record<string, number> = {
  rumor: 0,
  suspect: 1,
  assert: 2,
};
```

### §3 `selectReactionEvent`（T3）

```ts
export const MAX_REACTION_AGE_DAYS = 3;
export const MAX_REACTION_COOLDOWN_DAYS = 3;

export function selectReactionEvent(opts: {
  events: readonly CourtEvent[];
  speakerId: string;
  targetId: string;
  sceneDirective?: string;
  now: GameTime;
  presentCharacterIds: readonly string[];
  eventReactionLog: readonly EventReactionRecord[];
}): { event: CourtEvent; reactionContext: EventReactionContext } | undefined
```

**Eligibility（全部满足）：**

1. `eventToReactionContext(event, speakerId)` 非 `undefined`。
2. 时间窗：`0 ≤ now.dayIndex - event.occurredAt.dayIndex ≤ MAX_REACTION_AGE_DAYS`（未来事件不合格）。
3. Participant relevance：`event.participants.some(p => p.charId === speakerId || p.charId === targetId || presentCharacterIds.includes(p.charId))`。
4. 无冷却：`eventReactionLog` 中不存在 `{ speakerId, audienceId: targetId, eventId: event.id }` 且 `reactedAt.dayIndex >= now.dayIndex - MAX_REACTION_COOLDOWN_DAYS`。

**`sceneDirective` 存在 → 直接返回 `undefined`**（directive 主导本轮目标）。

**排序**（多个合格事件）：`occurredAt.dayIndex` 降序 → `publicSalience` 降序 → `id` 升序。

### §5 `resolveClaimSources` + `assembleClaims` 升级（T5）

```ts
// src/engine/dialogue/claimAssembler.ts（修改）

/**
 * Derives which memory/event refs authorize a given claim.
 * Returns [] if no source found — that claim cannot be included in authorizedClaims.
 */
export function resolveClaimSources(
  claim: DialogueClaim,
  memoryContext: MemoryContext,
  knownEvents: readonly CourtEvent[],
  sourceEventId?: string,   // 来自 buildReactionPlan，优先绑定 subject_event claim
): ContextRef[]

export interface ClaimAssemblyResult {
  allowed: AuthorizedClaim[];
  forbidden: AuthorizedClaim[];
}

// assembleClaims 签名更新（reactionPlan 改为可选）：
export function assembleClaims(args: {
  speakerId: string;
  builtReaction: BuiltReaction | undefined;   // undefined when no eligible event
  memoryContext: MemoryContext;
  beliefs: BeliefProjection;
  state: GameState;
  audience: DialogueAudienceContext;
}): ClaimAssemblyResult
```

**`resolveClaimSources` 规则**（按 predicate 分类）：

- `holds_rank(subjectId, rankId)`:
  - Memory: `memCtx.activatedMemories` 中 subject 含 `subjectId`、triggerTags 含 `"rank"` → `{ kind: "memory", id }`
  - Event: `knownEvents` 中 `rank_changed`、subject 为 `subjectId`、`payload.to === rankId` → `{ kind: "event", id }`
  - 若 `sourceEventId` 对应的事件是 `rank_changed` 且 subject 匹配 → 优先加入
- `resides_at(subjectId, locationId)`:
  - Memory: triggerTags 含 `"residence"`，subjectIds 含 `subjectId` → memory ref
  - Event: `residence_changed`，mover = subjectId，`payload.to === locationId` → event ref
- `alive(subjectId)`:
  - Memory: subjectIds 含 subjectId → memory ref
  - Event: `heir_born`，heirId = subjectId → event ref（生命事实）
- 其他 predicate：仅查 memory（无 event 映射）。
- **无 source → 该 claim 不进 allowed/forbidden 列表**（source-less claim 无法过 gate）。

**聚合规则（AuthorizedClaim 构造）**：

- 按 `claimFactKey` 聚合（不按 claim id），source refs 取 union，certainty ceiling 取最小值（最保守）。
- forbidden：`currently_same_residence` 以"禁止做出的断言"语义生成 `object: true`（如果 speaker/subject 实际不同住，则禁止 speaker 声称同住）。

### §6 Claim gate coverage rule（T6，正确文件）

```ts
// src/engine/dialogue/claimGate.ts（修改，不是 gates.ts）

// 新增违规码
type ClaimViolationCode =
  | ...existing...
  | "claim_not_allowed"
  | "claim_explicitly_forbidden"
  | "source_not_authorized";

/**
 * Coverage check: proposed claim is covered by an authorized claim when:
 * 1. fact keys match ([predicate, subjectId, object])
 * 2. polarity matches (deny vs assert treated as opposite polarity)
 * 3. proposed modality strength ≤ authorized modality strength
 * 4. proposed certainty ≤ certaintyCeiling (if set)
 */
function isCoveredByAuthorizedClaim(
  proposed: ProposedClaim,
  authorized: AuthorizedClaim,
): boolean

/**
 * Source check: proposed source refs must be a subset of
 * the authorized claim's sourceRefs ∩ offeredRefs.
 */
function isSourceAuthorized(
  proposedSourceRef: ContextRef,
  authorized: AuthorizedClaim,
  offeredRefs: ReadonlySet<string>,
): boolean
```

**Gate 执行顺序（在 `claimGate.ts` 现有检查后追加）**：

1. 若 proposed claim 覆盖任意 `policy.forbiddenClaims` → `claim_explicitly_forbidden`。
2. 若 `policy.allowedClaims` 非 undefined（即使 `[]`）且 proposed 无任何 coverage → `claim_not_allowed`。
3. 对每个 proposed `sourceContextId`，解析为 `ContextRef`；若解析后的 ref 不在该 claim 的 authorized sourceRefs ∩ offeredRefs → `source_not_authorized`。

`proposedClaim.sourceContextIds` 解析为 ContextRef（按前缀 `"event:"` 或 `"memory:"`，无前缀按 `"memory"` 处理以向后兼容）。

### §7 Retrieval 分离（T7，`buildMemoryContext`）

```ts

export function buildMemoryContext(
  db: ContentDB,
  state: GameState,
  speakerId: string,
```

- Recall memory 时，query 包含 `targetId` 和 `audienceCharacterIds`（提高相关 memory 得分）。

- Memory quota 不变（topMemories 个）。
- `MemoryContext.knownEvents` = 独立 event quota 的结果。
