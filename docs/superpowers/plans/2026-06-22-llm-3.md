## Global Constraints

1. **唯一装配点**：四字段只在 `assembleDialogueRequest` 填充；`compilePromptPayload` 纯透传。
2. **Typed `ContextRef` 全仓贯穿**：`{ kind: "memory" | "event" | "fact"; id: string }`。Provider contract（`claims.ts` schema）使用 Zod discriminated union；gate 无需字符串 parse；writeback 按 kind 分流。
3. **Fact key 不含 modality**：`claimFactKey = [predicate, subjectId, object ?? null]`（object 不为 null 时才存在）。Modality 另判。
4. **AuthorizedClaim 聚合 key 含 polarity**：`authorizedClaimAggKey = claimFactKey + ":" + claimPolarity`。不跨 polarity 聚合。
5. **Modality 单一来源 + polarity 分离**：gate 只读 `claim.modality`；schema superRefine 验证 `proposed.modality === proposed.claim.modality`；`polarity = modality === "deny" ? "deny" : "affirm"`；strength 只对 affirm 排序。
6. **Source refs 非空（factual claim）**：`sourceRefs.length >= 1`；schema `.min(1)`；空 refs → `source_not_authorized`。
7. **Empty allowedClaims = CLOSED**：`policy.allowedClaims` 非 undefined 时（含空数组），无覆盖 factual claim → `claim_not_allowed`。
8. **Reaction event 强制 pin 进 prompt**：`selectPromptEvents` 保证 `pinnedEventId` 恒在 `promptContext.knownEvents`，从而恒在 `offeredEventRefs`。
9. **Event claim 验当前 state**：`eventToAuthorizedClaims(event, state, sourceRef)` 先核验当前 state 仍与事件结果一致，否则跳过该 claim。
10. **Event once-only**：同 `(speakerId, audienceId, eventId)` 三元组只写一次；writeback guard 幂等。
11. **Scripted 不写 reaction log**：`request.scripted !== undefined` 时跳过 event reaction writeback。
12. **Writeback 原子**：memory mention + event reaction 在同一 state 替换中完成。
13. **`produceDialogueTurn` 统一入口**：generative 调用必须走 policy-aware pipeline；SceneRunner/ReactionScreen/BedchamberScene 全部接入；消费 `nextState`。
14. **Display 不泄漏 internal ID**：`resolvePromptEntityName` fallback 为 `"某人"`，rank display fallback 为 `"某位分"`，location display fallback 为 `"某处"`；结构化机器字段可保留 raw ID。
15. **正确文件路径**：claim gate → `claimGate.ts`；`ClaimViolationCode` → `claimGate.ts`；`DialoguePromptContext` / `PromptEvent` → `promptPayload.ts`；`ProposedClaim` schema → `claims.ts`；save → `saveSystem.ts`（`SAVE_FORMAT_VERSION = 8`）+ `stateSchema.ts`（字段 schema）。
16. **测试**：每个 task 结束须 `npx vitest run` 全绿 + `npx tsc --noEmit` clean。

// src/engine/dialogue/types.ts

export interface ContextRef {
  kind: "memory" | "event" | "fact";   // "fact" reserved for LLM-4+
  id: string;
}
export const contextRefKey = (r: ContextRef): string => `${r.kind}:${r.id}`;

// Fact key: no modality, no claim id. object may be absent.
export const claimFactKey = (
  c: Pick<DialogueClaim, "predicate" | "subjectId"> & { object?: unknown },
): string => JSON.stringify([c.predicate, c.subjectId, c.object ?? null]);

export type ClaimPolarity = "affirm" | "deny";
export const claimPolarity = (m: ClaimModality): ClaimPolarity =>
  m === "deny" ? "deny" : "affirm";

export const MODALITY_STRENGTH: Partial<Record<ClaimModality, number>> = {
  rumor: 0, suspect: 1, assert: 2,
};

// Aggregation key includes polarity to prevent merging affirm + deny
export const authorizedClaimAggKey = (c: DialogueClaim): string =>
  `${claimFactKey(c)}:${claimPolarity(c.modality)}`;

export interface AuthorizedClaim {
  claim: DialogueClaim;      // claim.certaintyCeiling (0–100) used directly
  sourceRefs: ContextRef[];  // non-empty
}

  offeredMemoryRefs: ReadonlySet<string>;
  offeredEventRefs: ReadonlySet<string>;

}

解析顺序：

### §3 `eventToAuthorizedClaims(event, state, sourceRef)`（T6）

```ts
// src/engine/dialogue/claimAssembler.ts

export function eventToAuthorizedClaims(
  event: CourtEvent,
  state: GameState,
  sourceRef: ContextRef,   // { kind: "event", id: event.id }
): AuthorizedClaim[]
```

| `event.type` | 生成 claim | state 验证 | 无 object |
|---|---|---|---|
| `rank_changed` | `holds_rank(subjectId, payload.to)` assert | `state.standing[subjectId]?.rank === payload.to` | — |
| `residence_changed` | `resides_at(moverId, payload.to)` assert | `state.standing[moverId]?.residence === payload.to` | — |
| `heir_born` | `alive(heirId)` assert | heir 存在且 lifecycle === `"alive"` | object 省略 |
| `heir_died` | `alive(heirId)` deny | heir 存在且 lifecycle !== `"alive"` | object 省略 |
| 其余 | `[]` | — | — |

State 验证失败 → 跳过该 claim（不 crash）。

Claim ID 格式（确定性）：
```ts
id: `event:${event.id}:${claim.predicate}:${claim.subjectId}:${claimPolarity(claim.modality)}`
```
