## Global Constraints

1. **唯一公开入口**：只导出 `produceDialogueTurn`；`produceDialogueLine` / `produceDialogueLineWithPolicy` 降为文件私有。
2. **Gate 分支只看 `provider.kind`**：`"scripted"` → speaker + text gate；`"generative"` → 完整 policy pipeline。`generative + scripted request` → error。
3. **typed `ContextRef` 全仓贯穿**：`{ kind: "memory" | "event" | "fact"; id: string }`。T7 做原子迁移，完成后 `grep sourceContextIds → 0`。
4. **Event auth 替代 belief knowledge**：`allowedClaims` 覆盖某 claim → 跳过 `reveals_unknown_fact`；BeliefProjection 返回明确相反值仍触发 `contradicts_known_fact`。
5. **`isLatestFactMutation` 守卫**：`eventToAuthorizedClaims` 双重验证：state check + latest-mutation check。
6. **AuthorizedClaim 聚合 key 含 polarity**：`authorizedClaimAggKey = claimFactKey + ":" + claimPolarity`，不跨 polarity 聚合。
7. **Source refs 非空（factual claim）**：schema `.min(1)`；空 refs → `source_not_authorized`。
8. **Empty allowedClaims = CLOSED**：`allowedClaims` 非 undefined 且无覆盖 → `claim_not_allowed`。
9. **Reaction event 强制 pin**：`selectPromptEvents` 有 `pinnedEventId` 时，若 `pinnedEventId` 不在 events 或 `limit < 1` → throw（not silent failure）。
10. **SceneRunner 双状态**：`preState` 用于 branch 条件；`dialogueState` 用于 request 装配；每行成功后 `dialogueState = nextState`。
11. **CAS commit**：UI/store 调用 `GameStore.commitDialogueState(expectedState, nextState)`。
12. **Abandon Option A**：SceneRunner 注释、DialogueScreen quit 文案、abandon 测试全部同步。
13. **Writeback 原子 + 幂等**：mention + reaction 同一 state 替换；guard 防重复写入。
14. **Scripted provider 不写 reaction log**：`provider.kind === "scripted"` 时跳过 event reaction writeback。
15. **`isLatestFactMutation` 顺序**：同 predicate + subject 的所有 chronicle 事件按 `occurredAt.dayIndex desc, id desc` 排序；当前事件不在首位 → 跳过。
16. **`initialState.ts` + `newGame.ts`** 都初始化 `eventReactionLog: []`。
17. **文件路径**：claim gate → `claimGate.ts`；`ClaimViolationCode` → `claimGate.ts`；`DialoguePromptContext`/`PromptEvent` → `promptPayload.ts`；`ProposedClaim` schema → `claims.ts`；save → `saveSystem.ts`（`SAVE_FORMAT_VERSION = 8`）+ `stateSchema.ts`。
18. **测试**：每个 task 结束须 `npx vitest run` 全绿 + `npx tsc --noEmit` clean。

### §1 `isLatestFactMutation`（T6）

```ts
// src/engine/dialogue/claimAssembler.ts

/**
 * Returns true only if `event` is the latest mutation of the same
 * predicate/subject pair in the chronicle.
 */
export function isLatestFactMutation(
  event: CourtEvent,
  chronicle: readonly CourtEvent[],
): boolean
```

Fact mutation 定义（predicate × subjectId）：
- `rank_changed` → `holds_rank` × `participant("subject")`
- `residence_changed` → `resides_at` × `participant("mover")`
- `heir_born` / `heir_died` → `alive` × `payload.heirId`（两类共享同一 predicate × subject）

排序：`occurredAt.dayIndex desc, event.id desc`（字典序）。`event` 不在首位 → `false`。

### §2 `eventToAuthorizedClaims(event, state, sourceRef)`（T6 核心）

双重验证：state check + latest-mutation check。两者都通过才生成 claim。

| `event.type` | state 验证 | latest-mutation | claim（object 省略时不写字段）|
|---|---|---|---|
| `rank_changed` | `state.standing[subjectId]?.rank === payload.to` | `isLatestFactMutation` | `holds_rank(subjectId, payload.to)` assert |
| `residence_changed` | `state.standing[moverId]?.residence === payload.to` | 同上 | `resides_at(moverId, payload.to)` assert |
| `heir_born` | `heir.lifecycle === "alive"` | 同上 | `alive(heirId)` assert（无 object） |
| `heir_died` | `heir.lifecycle !== "alive"` | 同上 | `alive(heirId)` deny（无 object） |
| 其余 | — | — | `[]` |

Claim ID：`event:${event.id}:${predicate}:${subjectId}:${polarity}`

### §3 Claim gate 执行顺序（T7，claimGate.ts）

```
1. sourceRefs.length === 0 → source_not_authorized（fail fast）
2. forbiddenClaims 覆盖 → claim_explicitly_forbidden
3. allowedClaims !== undefined:
   a. 找 matchingAuthorized（fact key + polarity + modality strength ≤）
   b. 无 match → claim_not_allowed
   c. 有 match → 验每个 proposed sourceRef ∈ matchingAuthorized.sourceRefs ∩ offeredRefs
                  不满足 → source_not_authorized
   d. 有 match 且 source ok → eventAuthorized = true
4. eventAuthorized = true:
   - 跳过 reveals_unknown_fact
   - BeliefProjection 返回明确相反值 → contradicts_known_fact（仍检查）
5. eventAuthorized = false（allowedClaims === undefined, backward compat）:
   - 正常 reveals_unknown_fact / contradicts_known_fact 检查
```
