## Global Constraints

1. **SceneRunner 永远 scripted-only**：SceneRunner 传入的 provider 必须是 scripted；所有 line node 都设 `request.scripted`；不写 mention/reaction 记录；existing abandon contract 不变。
2. **Generative 流独立**：`DialogueScreen` generative mode 不经 SceneRunner；`provider.kind === "generative"`；request 无 `scripted`；全 policy pipeline；CAS commit。
3. **唯一公开入口**：只导出 `produceDialogueTurn`；`produceDialogueLine` / `produceDialogueLineWithPolicy` 降为文件私有。
4. **Gate 分支只看 `provider.kind`**：`"scripted"` → speaker + text gate；`"generative"` → 全 policy pipeline；`generative + request.scripted → error invalid_combination`。
5. **CAS 协议**：generative turn 调用 `produceDialogueTurn` 前保存 `expectedState = store.state`；CAS 成功后才展示台词 + 更新本地状态；失败时 abort（不展示、不更新）。
6. **Assembly 顺序**：`selectPromptEvents` → `assembleClaims(offeredEvents = promptEvents)` → `promptContext`。Assembler 只从 `offeredEvents` 产 event claims；memory→event 链也只在 `offeredEvents` 内验证。
7. **`forbiddenClaims: readonly DialogueClaim[]`**：`isCoveredByForbiddenClaim` 只检 fact key + polarity（不检 source、modality strength、certainty）。
8. **`contradicts_speaker_belief`**：使用现有名称；新增 `isContradictedByBelief(claim, believed)` helper 统一 binary + object predicate 语义。
9. **typed `ContextRef` 全仓贯穿**：T7 做原子迁移，完成后 `grep sourceContextIds → 0`。
10. **AuthorizedClaim 聚合 key 含 polarity**：不跨 polarity 聚合。
11. **Source refs 非空**：schema `.min(1)`；空 refs → `source_not_authorized`。
12. **Empty allowedClaims = CLOSED**：`allowedClaims` 非 undefined 且无覆盖 → `claim_not_allowed`。
13. **Event auth 替代 belief knowledge**：`allowedClaims` 覆盖 → 跳过 `reveals_unknown_fact`；contradiction 仍检查。
14. **`isLatestFactMutation` 守卫**：double check：state match + latest-mutation。
15. **Reaction event pin 强制**：`selectPromptEvents` throw when precondition violated。
16. **Writeback 原子 + 幂等**；scripted provider 不写 reaction。
17. **测试**：每个 task 结束须 `npx vitest run` 全绿 + `npx tsc --noEmit` clean。

### §1 `isLatestFactMutation`（T6）

Fact mutation 定义（predicate × subjectId）：
- `rank_changed` → `holds_rank` × `participant("subject")`
- `residence_changed` → `resides_at` × `participant("mover")`
- `heir_born` / `heir_died` → `alive` × `payload.heirId`

排序：`occurredAt.dayIndex desc, event.id desc`（字典序）。`event` 不在首位 → `false`。

### §2 `eventToAuthorizedClaims(event, state, sourceRef)`（T6）

双重验证：state check + `isLatestFactMutation`。

| `event.type` | state 验证 | claim |
|---|---|---|
| `rank_changed` | `state.standing[subjectId]?.rank === payload.to` | `holds_rank(subjectId, payload.to)` assert |
| `residence_changed` | `state.standing[moverId]?.residence === payload.to` | `resides_at(moverId, payload.to)` assert |
| `heir_born` | `heir.lifecycle === "alive"` | `alive(heirId)` assert（无 object） |
| `heir_died` | `heir.lifecycle !== "alive"` | `alive(heirId)` deny（无 object） |

Claim ID：`event:${event.id}:${predicate}:${subjectId}:${polarity}`

### §3 `isContradictedByBelief` + Claim gate 顺序（T7）

```ts
// src/engine/dialogue/claimGate.ts（新增 helper）

/**
 * Returns true if the proposed claim conflicts with a known belief value.
 * For binary predicates (alive, no object): affirm expects true, deny expects false.
 * For object predicates (holds_rank, resides_at): affirm expects object match, deny expects mismatch.
 */
export function isContradictedByBelief(
  claim: DialogueClaim,
  believed: string | boolean,
): boolean {
  const polarity = claimPolarity(claim.modality);
  if (claim.object === undefined) {
    return polarity === "affirm" ? believed === false : believed === true;
  }
  return polarity === "affirm"
    ? believed !== claim.object
    : believed === claim.object;
}
```

Gate 执行顺序：
```
1. sourceRefs.length === 0 → source_not_authorized
2. forbiddenClaims 覆盖（isCoveredByForbiddenClaim：只查 fact key + polarity）→ claim_explicitly_forbidden
3. allowedClaims !== undefined:
   a. 找 matchingAuthorized（fact key + polarity + modality strength ≤ + certainty ≤ ceiling）
   b. 无 match → claim_not_allowed
   c. 有 match → 验每个 sourceRef ∈ matchingAuthorized.sourceRefs ∩ offeredRefs
                  不满足 → source_not_authorized
   d. 有 match 且 source ok → eventAuthorized = true
4. eventAuthorized = true:
   - 跳过 reveals_unknown_fact
   - BeliefProjection 返回值 → isContradictedByBelief → contradicts_speaker_belief（仍检查）
5. eventAuthorized = false（allowedClaims === undefined，backward compat）:
   - 正常 reveals_unknown_fact / contradicts_speaker_belief 检查
```

新 `ClaimViolationCode`（加到 `claimGate.ts`）：
```ts
| "claim_not_allowed"
| "claim_explicitly_forbidden"
| "source_not_authorized"
```

### §4 `assembleClaims` 接 `offeredEvents`（T6 + T8）

```ts
export function assembleClaims(args: {
  speakerId: string;
  builtReaction: BuiltReaction | undefined;
  offeredMemories: readonly ActivatedMemory[];     // = memCtx.activatedMemories
  offeredEvents: readonly CourtEvent[];             // = promptEvents（已由 selectPromptEvents 选定）

  state: GameState;

}): ClaimAssemblyResult
```

Event claim 来源仅为 `offeredEvents`（不是 `knownEventsAll`）：
```ts
// allowed claims:
for (const event of offeredEvents) {
  const claims = eventToAuthorizedClaims(event, state, { kind: "event", id: event.id });
  claims.forEach(c => addToAllowed(c));
}

// memory→event chain（memory 和其 sourceEventId event 均须在 offered 内）:
for (const memory of offeredMemories) {
  if (!memory.sourceEventId) continue;
  const sourceEvent = offeredEvents.find(e => e.id === memory.sourceEventId);
  if (!sourceEvent) continue;                    // event not offered → skip
  const claims = eventToAuthorizedClaims(sourceEvent, state, { kind: "memory", id: memory.id });
  claims.forEach(c => addToAllowed(c));
}
```

Assembler 不生成 `sourceRefs` 为空的 claim（内部 guard）。

### §6 Runtime 模式（T9）

```
┌─────────────────────────────────────────────────────────────────┐
│  Authored scene (SceneRunner)                                   │
│  provider: ScriptedDialogueProvider (always)                    │
│  request.scripted = { text: node.text }                         │
│  gate: speaker + text only                                      │
│  writeback: NONE (scripted doesn't write records)               │
│  abandon: no records = no consequences (existing contract)      │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  Generative dialogue (DialogueScreen free-conversation mode)    │
│  provider: dialogueProvider (injected via App.tsx from main.tsx)│
│  request.scripted: undefined                                    │
│  gate: full policy pipeline                                     │
│  writeback: mention + reaction records on CAS success           │
│  abandon: already-committed records stay (Option A)             │
│  fallback: dialogueProvider === undefined → buildConversation() │
└─────────────────────────────────────────────────────────────────┘
```

CAS 协议（`DialogueScreen`）：
```ts
const runGenerativeTurn = async () => {
  const expectedState = store.state;                     // snapshot before async
  const request = assembleDialogueRequest(db, expectedState, speakerId, locationId, {
    targetId: activeConversationTarget,
  });

  const result = await produceDialogueTurn(db, dialogueProvider, request, expectedState);
  if (!result.ok) { handleError(result.error); return; }

  const committed = store.commitDialogueState(expectedState, result.value.nextState);
  if (!committed) {
    // state changed during LLM call — abort (don't display stale line)
    setError("DIALOGUE_STATE_STALE");
    return;
  }

  displayLine(result.value.line);
};
```
