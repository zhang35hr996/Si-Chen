## Global Constraints

1. **唯一装配点**：四字段只在 `assembleDialogueRequest` 填充；`compilePromptPayload` 纯透传零改动。
2. **typed `ContextRef` 贯穿**：从 `AuthorizedClaim.sourceRefs` 到 `ProposedClaim.sourceRefs`，统一 `{ kind: "memory"|"event"; id: string }`。Gate 无需字符串 parse；writeback 按 `kind` 分流。
3. **Fact key 不含 modality**：`claimFactKey = [predicate, subjectId, object ?? null]`。Modality 和 polarity 另判。
4. **单一 modality 来源**：gate 只读 `claim.modality`；Zod schema 要求 `proposed.modality === proposed.claim.modality`。
5. **`deny` = 极性，不是弱模态**：`polarity = modality === "deny" ? "deny" : "affirm"`。object 不取反。Coverage：affm ≠ deny；deny only covers deny。Modality strength（`rumor<suspect<assert`）只适用于 affirm 极性。
6. **Empty allowedClaims = CLOSED**：`allowedClaims: AuthorizedClaim[]`（非 undefined）时，任何无覆盖的 factual proposed claim → `claim_not_allowed`，即使列表为空。
7. **Source refs 非空**：factual proposed claim 的 `sourceRefs` 须 `length >= 1`（schema `.min(1)`）；空 refs → `source_not_authorized`。
8. **Event provenance 专属链路**：claim 的 event source 只来自 `eventToAuthorizedClaims(event)` 的输出；memory 的事实来源只通过 `memory.sourceEventId` 链路（sourceEvent → `eventToAuthorizedClaims`）。禁止按 `triggerTags` 猜事实。
9. **Event once-only**：同一 `(speakerId, audienceId, eventId)` 三元组只写一次 `EventReactionRecord`。无 cooldown days。
10. **Eligibility 先于 quota**：`buildMemoryContext` 拿到所有可知 events，先 eligibility 过滤，再按 quota 截取。
11. **Retrieval 包含 speakerId**：`RecallQuery.subjectIds` 必须包含 `speakerId`（防止 speaker 自身事件被遗漏）。
12. **原子 state 更新**：memory mention writeback 和 event reaction writeback 须在同一次 state 替换中完成。
13. **Scripted 不写 reaction log**：`request.scripted !== undefined` 时跳过 event reaction writeback。
14. **文件正确性**：claim gate → `claimGate.ts`；`DialoguePromptContext` → `promptPayload.ts`；`ClaimViolationCode` → `claimGate.ts`；save schema → `saveSystem.ts`（`SAVE_FORMAT_VERSION=8`）+ `stateSchema.ts`（字段 schema）。
15. **`state.date` → `toGameTime(state.calendar)`**：全文统一。
16. **测试**：每个 task 结束须 `npx vitest run` 全绿 + `npx tsc --noEmit` clean。

### §0 核心类型（T5 定义，全局共用）

```ts
// src/engine/dialogue/types.ts（追加）

export interface ContextRef {
  kind: "memory" | "event";
  id: string;
}

/** Stable de-dup key for ContextRef in maps/sets. */
export const contextRefKey = (r: ContextRef): string => `${r.kind}:${r.id}`;

/**
 * Fact key — predicate + subjectId + object only. No modality, no claim id.
 * Used for semantic matching between proposed and authorized claims.
 */
export const claimFactKey = (
  c: Pick<DialogueClaim, "predicate" | "subjectId" | "object">,
): string => JSON.stringify([c.predicate, c.subjectId, c.object ?? null]);

/** Polarity derived from modality. deny = opposite polarity; others = affirm. */
export type ClaimPolarity = "affirm" | "deny";
export const claimPolarity = (modality: ClaimModality): ClaimPolarity =>
  modality === "deny" ? "deny" : "affirm";

/** Modality strength for affirm-polarity modalities only. deny excluded. */
export const MODALITY_STRENGTH: Partial<Record<ClaimModality, number>> = {
  rumor: 0, suspect: 1, assert: 2,
};

/**
 * A claim the engine explicitly authorizes (allowed or forbidden).
 * sourceRefs lists which memory/event contexts authorize this claim.
 * No certaintyCeiling here — read claim.certaintyCeiling (0–100) directly.
 */
export interface AuthorizedClaim {
  claim: DialogueClaim;     // claim.certaintyCeiling already carries 0–100 ceiling
  sourceRefs: ContextRef[]; // non-empty: at least one authorizing context
}
```

  eventToReactionContext(e, opts.speakerId) !== undefined
  && e.occurredAt.dayIndex <= opts.now.dayIndex                           // 非未来事件
  && opts.now.dayIndex - e.occurredAt.dayIndex <= MAX_REACTION_AGE_DAYS  // 时间窗

### §7 `assembleClaims` 升级（T5）

```ts
// src/engine/dialogue/claimAssembler.ts

export interface ClaimAssemblyResult {
  allowed: AuthorizedClaim[];
  forbidden: AuthorizedClaim[];
}

export function assembleClaims(args: {
  speakerId: string;
  builtReaction: BuiltReaction | undefined;
  memoryContext: MemoryContext;
  beliefs: BeliefProjection;
  state: GameState;
  audience: DialogueAudienceContext;
}): ClaimAssemblyResult
```

**Source binding 规则：**

- **Event-sourced claims**：`builtReaction?.sourceEventId` 对应 event 存在于 `memoryContext.knownEventsAll`（全量，非 quota 后）时，调用 `eventToAuthorizedClaims(event, { kind: "event", id: event.id })`，结果纳入 `allowed`。
- **Memory-sourced claims**：`memory.sourceEventId` 指向一个本轮可知并已提供给模型的 event → 调用 `eventToAuthorizedClaims(sourceEvent, ...)` 看此 memory 是否"见证"了某 claim → 若是，以 memory ref 作为 source 额外授权（`sourceRefs` 加 memory ref）。**禁止**按 `triggerTags` 猜事实。
- **无结构化 source 的初始 authored memory**：不生成 fact claim（`sourceEventId` 为空 → 跳过）。
- **Forbidden**：只从当前游戏状态派生，不依赖 memory tag：
  - `currently_same_residence = true`（object: true）：若 speaker 有旧「共同居所」记忆（memory.kind 或 memory.subjectIds 含 target），但当前 `state.standing[speakerId]?.residence !== state.standing[targetId]?.residence` → 生成 `{ claim: { predicate: "currently_same_residence", subjectId: targetId, object: true, modality: "assert" }, sourceRefs: [<该 memory ref>] }`。

**聚合：** 按 `claimFactKey` 聚合，`sourceRefs` 取 union（去重），`certaintyCeiling` 取最小值。结果 `sourceRefs` 非空才入列表。

### §8 Claim gate coverage rule（T6，正确文件）

```ts
// src/engine/dialogue/claimGate.ts（修改，非 gates.ts）

// 新违规码（追加到 ClaimViolationCode，ClaimViolationCode 也定义在此文件）：
// | "claim_not_allowed"
// | "claim_explicitly_forbidden"
// | "source_not_authorized"

function isCoveredByAuthorizedClaim(
  proposed: ProposedClaim,   // proposed.claim.modality 已经过 schema 验证 = proposed.modality
  authorized: AuthorizedClaim,
): boolean {
  // 1. fact key match
  if (claimFactKey(proposed.claim) !== claimFactKey(authorized.claim)) return false;
  // 2. polarity match
  if (claimPolarity(proposed.claim.modality) !== claimPolarity(authorized.claim.modality)) return false;
  // 3. modality strength (affirm only): proposed ≤ authorized
  const ps = MODALITY_STRENGTH[proposed.claim.modality];
  const as = MODALITY_STRENGTH[authorized.claim.modality];
  if (ps !== undefined && as !== undefined && ps > as) return false;
  // 4. certainty ceiling (0–100)
  const ceiling = authorized.claim.certaintyCeiling;
  if (ceiling !== undefined && (proposed.certainty ?? 100) > ceiling) return false;
  return true;
}
```

**Gate 执行顺序（追加在现有 claimGate 检查后）：**

1. Proposed `sourceRefs.length === 0` → `source_not_authorized`（即使 claim 本身有覆盖）。
2. 若 proposed claim 被任一 `policy.forbiddenClaims` 覆盖 → `claim_explicitly_forbidden`。
3. 若 `policy.allowedClaims` 非 undefined 且无 coverage → `claim_not_allowed`（空数组也执行）。
4. 对每个 proposed `sourceRef`：`ref` 不在 `{ ...authorized.sourceRefs }` ∩ `offeredRefs` → `source_not_authorized`。

> Gate function 参数 `policy.allowedClaims?: readonly AuthorizedClaim[]`（可选，undefined 时步骤 3 跳过，向后兼容现有独立单测）。完整 policy 装配后该字段非 undefined（即使空数组）。

## 验收标准

- [ ] `reactionPlan`: 无事件/有 sceneDirective → `undefined`；有合格事件 → 真实 plan
- [ ] `knownEvents`: whitelist payload；participant + rank/location displayName；无内部 ID
